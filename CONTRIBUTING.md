## Contribution Guidelines

While we appreciate community interest, please note:

1. **Architecture Decisions**: All major architectural decisions are made by the BDFL to maintain project vision and consistency.

2. **Pull Requests**: 
   - Bug fixes and minor improvements are welcome
   - Major features should be discussed in an issue first
   - All PRs require BDFL approval

3. **Issues**: 
   - Report bugs with clear reproduction steps
   - Feature requests are considered but not guaranteed

4. **Code Style**:
   - Follow existing patterns in the codebase
   - No unnecessary dependencies
   - Maintain O(1) complexity where possible
